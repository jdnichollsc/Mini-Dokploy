# Agent guide — Mini-Dokploy

Canonical instructions for any AI coding agent (Claude Code, Cursor, Codex,
Aider, Continue, etc.) working in this repository. Keep this file authoritative
and short — link to deeper docs (`README.md`, `docs/PLAN.md`) instead of
re-explaining them.

## What this project is

A small, locally-runnable PaaS: paste a Git URL + Dockerfile path, get a
reachable URL on `app-<id>.127.0.0.1.sslip.io`. Runs Docker Swarm + Traefik +
Temporal under the hood. Pages-Router Next.js + tRPC UI.
Plan + trade-offs: [`docs/PLAN.md`](./docs/PLAN.md).
Quick onboarding: [`README.md`](./README.md).

## Stack (don't change these without a strong reason)

- **Next.js Pages Router** (`apps/web/src/pages/`). The brief required it.
- **tRPC v11** with `@trpc/react-query` (NOT `@trpc/next` — TS 5.x + v11 had a
  type-inference regression with the latter).
- **BetterAuth** with the `organization` plugin (multi-tenant).
- **Temporal** for the deploy lifecycle (workflow runs in `apps/worker`,
  Temporal infra in `docker-compose.yml`).
- **Docker Swarm** for user deployments + Traefik + (in `pnpm stack:up` mode)
  the mini-dokploy web/worker.
- **Drizzle + libsql/SQLite** for app state (`packages/db`).
- **Vercel AI SDK + Anthropic** for the optional AI activities.
- **Pnpm 10 + Turborepo**.

## Hard rules (load-bearing, don't break)

1. **Package boundaries** are the safety mechanism. See the table in
   [`docs/PLAN.md` §3](./docs/PLAN.md). Specifically:
   - `packages/core` is **pure** — no I/O, no Node-only deps. Only `zod` and
     `nanoid` allowed.
   - `packages/api` (tRPC) must **never** import `dockerode`, `simple-git`,
     or `@ai-sdk/*`. Use `packages/workflow-client` to start workflows; use
     `packages/ai` for AI calls.
   - `packages/orchestrator` is **worker-only**. Anything that touches the
     Docker socket, runs `git clone`, or spawns `docker buildx` lives here.
   - `packages/db` only depends on `@mini-dokploy/env/db` (not `/server`). So
     the worker doesn't inherit BetterAuth's env requirements.
2. **Workflows are deterministic.** `apps/worker/src/workflows/*.ts` may only
   use `proxyActivities` + workflow-safe primitives (`sleep`, `condition`,
   signals, queries). Never import `fs`, `dockerode`, network, `Date.now()`,
   etc. into a workflow file.
3. **Traefik labels** for user services go through `buildTraefikLabels()` in
   `packages/core/src/traefik/labels.ts`. The hardened filter rejects
   cross-tenant, own-router-hijack, and own-service-rebind attempts. Don't
   sidestep it. Any new generated label key must be added to the reserved-
   suffix list **and** covered by a test in `labels.test.ts`.
4. **Deployment IDs** must be DNS-label-safe — use
   `generateDnsLabelSafeId()` from `packages/core/src/ids.ts`. The id appears
   in `app-<id>.sslip.io` hostnames, Swarm service names, and Traefik router
   names.
5. **Env loading** is centralised in `packages/env/src/load-env.ts`. It walks
   up to find the repo-root `.env` and anchors relative paths
   (`DATABASE_URL`, `DOKPLOY_LOG_DIR`, `DOKPLOY_BUILD_DIR`) to that root, so
   every process points at the same files no matter where it's launched
   from. Don't bypass it with `import 'dotenv/config'` — that breaks cwd
   anchoring.
6. **One DATABASE_URL.** SQLite lives at `./local.db` (repo root). If you
   see a stray `apps/*/local.db` someone bypassed load-env — fix the cause.
7. **Status state machine.** Deployment status enum is in
   `packages/core/src/status.ts`. Anywhere you write `deployment.status` or
   `deployment_run.status`, the value must come from there. Don't introduce
   ad-hoc strings.
8. **Migrations** are versioned SQL files generated by `drizzle-kit` in
   `packages/db/src/migrations/`. After changing a schema file under
   `packages/db/src/schema/`, run `pnpm db:generate` and commit the new
   migration. The web container applies pending migrations at startup
   (`apps/web/docker-entrypoint.sh`).
9. **Tests must stay green.** `pnpm -F @mini-dokploy/core test` and
   `pnpm test:e2e` are part of the contract.

## Commands you'll use most

```sh
pnpm dev                                  # boot Temporal + Swarm + Traefik + web + worker
pnpm -F @mini-dokploy/core test           # 10 unit tests (Traefik labels)
pnpm test:e2e                             # Playwright smoke (~13s)
pnpm test:e2e:full                        # full deploy lifecycle (needs pnpm stack:up)
pnpm -F web exec tsc --noEmit             # typecheck web
pnpm -F worker check-types                # typecheck worker
pnpm -F @mini-dokploy/api exec tsc --noEmit
pnpm db:generate                          # after editing a schema file
pnpm db:studio                            # GUI against ./local.db
pnpm stack:up / stack:down                # full containerised mode
pnpm temporal:down                        # stop Temporal compose
```

## Where to look at runtime

- **Mini-Dokploy UI:** http://localhost:3001
- **Temporal UI** (workflow history, retries, payloads): http://localhost:8080
- **Traefik dashboard** (live routers/services): http://localhost:8081/dashboard/
- **Build logs (per run):** `./.dokploy-logs/<runId>.log`
- **Swarm services:** `docker service ls`, `docker service ps app-<id>`
- **Database GUI:** `pnpm db:studio`

## Gotchas already paid for (don't re-discover them)

- **Newer Docker engines reject Traefik's hardcoded `/v1.24/*` API paths.**
  Fixed by `scripts/socket-proxy.conf` (nginx rewrites to `/v1.40/*`). Don't
  remove the proxy — Traefik breaks without it on Docker engine 29+.
- **Docker Desktop port forwarding sometimes wedges.** Symptoms: TCP
  connects work but HTTP/gRPC handshakes hang. Fix: quit + reopen Docker
  Desktop. (Not our code.)
- **Stale `apps/*/.env` files** override our root `.env` because Next.js
  auto-loads them. `load-env.ts` uses `override: true` to win, but it's
  safer to keep only one `.env` at the repo root.
- **Relative `file:./local.db`** resolves per-process cwd. Always rely on
  `load-env.ts` to anchor — or pass an absolute path explicitly.
- **`dokku/smoke-test-app`** is buildpack-only, not Dockerfile. Don't
  suggest it. Use `docker/welcome-to-docker` (port 3000, branch `main`) or
  any repo with a real `Dockerfile`.
- **Default branch matters.** Many older test repos default to `master`.
  Fill the form field explicitly; otherwise `git clone --branch=main`
  fails with `Remote branch main not found`.

## When you change things

- **Touching `apps/worker/src/workflows/*`:** keep deterministic. Run
  `pnpm -F worker check-types`. If you add a new activity, also register it
  in `apps/worker/src/activities/index.ts`.
- **Touching `packages/core/src/traefik/labels.ts`:** add tests in
  `labels.test.ts` covering the new behavior, especially for tenant safety.
- **Touching `packages/db/src/schema/*`:** run `pnpm db:generate`, commit
  the new file under `src/migrations/`.
- **Touching `packages/api/src/routers/*`:** make sure protected routes use
  `orgScopedProcedure` (defined in `packages/api/src/index.ts`) — never
  `protectedProcedure` directly for tenant-scoped reads/writes.
- **Touching `scripts/bootstrap.sh` or any `scripts/*.sh`:** keep them
  idempotent. They run on every `pnpm dev`.

## What this project intentionally does NOT do

- Build with anything other than a Dockerfile (no nixpacks, no buildpacks).
- Multi-node Swarm (single-node manager assumption).
- Multi-replica web/worker (file-tailed WebSocket logs assume one writer).
- Push images to a registry (built locally via `docker buildx --load`).
- Persist Temporal across `pnpm stack:down` resets unless `temporal_postgres_data` volume is kept.

See "What I'd build next" in `README.md` for the post-MVP roadmap.

```sh
pnpm -F @mini-dokploy/core test          # unit tests
pnpm -F web exec tsc --noEmit            # web typecheck
pnpm -F worker check-types               # worker typecheck
pnpm -F @mini-dokploy/api exec tsc --noEmit

```

All five green → ship.
